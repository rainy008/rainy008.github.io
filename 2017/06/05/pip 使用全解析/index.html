python3.4以后自带pip，无需再安装。

1、使用国内安装源

​ 用pip安装依赖包时默认访问https://pypi.Python.org/simple/，但是经常出现不稳定以及访问速度非常慢的情况，国内厂商提供的pipy镜像速度快、稳定，目前可用的有：

​ 清华：https://pypi.tuna.tsinghua.edu.cn/simple

​ 阿里云：http://mirrors.aliyun.com/pypi/simple/

​ 中国科技大学 https://pypi.mirrors.ustc.edu.cn/simple/

​ 华中理工大学：http://pypi.hustunique.com/

​ 山东理工大学：http://pypi.sdutlinux.org/

​ 豆瓣：http://pypi.douban.com/simple/

5、pip离线使用

当不能联网而使用python时，你会遇到各种关于安装、使用包的坑，你会疑惑、沮丧、痛苦，甚至绝望……

​ 直到有一天，你突然知道pip原来可以离线安装包……

​ 你发现原来大神就是大神，创造python的大神们早已为你准备好了一切，只是你自己一直徘徊在单机的苦难中，没有找到通往“便捷”的天堂之道……

​ 你又重拾起沿着python之路一往无前的勇气，向着人机自由交流的光明未来继续前行……

​ 以上就是我近半年来的经历！也是我要好好总结pip使用的原因！

​ 首先，在一台能联网的电脑上更新要下载的包

​ 然后，在你喜欢的地方创建一个文件夹用来存放要下载的包，例如 D:\packages\

​ 接着，运行命令：pip freeze > D:\requirements.txt (或者 req.txt 随你喜欢的文件名),你安装的包的名称、版本号都会被导入D盘requirems.txt文件内，内容如下：

​ 之后，运行命令pip download -d d:\packages -r d:\requirements.txt，下载上述所列包。

​ 最后，将D:\packages文件夹中的包拷贝到不能联网的电脑上，运行命令pip install install --no-index --find-links=d:\packages -r requirements.txt，requirements.txt文件中所列的包就安装到未联网的电脑上了

​ 从此，再不用痛苦包的问题了！！！

关于 download 和 install 命令的具体用法，现用现help :-)
